# SNS_OPERATION.md — 宝塚Today X（旧Twitter）運用基盤

このファイルは公式Xアカウント（[@TakaTodayJP](https://x.com/TakaTodayJP)、2026-06-23開設）の運用方針・KPI記録・月次レビューを一元管理するための台帳。
フォロー中・フォローバックのアカウント管理は [SNS_ACCOUNT_LIST.md](./SNS_ACCOUNT_LIST.md) を参照。
記事のSNS文面生成ルールそのものは [PROJECT_RULES.md](../PROJECT_RULES.md) の「SNS運用方針」が優先（本ファイルと矛盾する場合はPROJECT_RULES.mdに従う）。

---

## 1. 運用目的

- 地域密着メディアとして認知される
- 地域住民との接点を増やす
- サイト（takarazuka-today.jp）への流入を増やす
- 行政・防災情報を届ける
- 地域イベント情報を拡散する
- 情報提供ネットワークを作る

---

## 2. 投稿ジャンル

- 防災
- 市議会
- 財政ウォッチ
- 行政情報
- イベント
- ライブカメラ
- 喫煙所
- 高校受験
- 子育て
- 交通
- 阪神競馬場（地域情報のみ。レース予想・オッズ・払戻金等は対象外。[PROJECT_RULES.md](../PROJECT_RULES.md)の阪神競馬場運営ルールに従う）

## 3. 投稿頻度

| 区分 | 頻度 |
|---|---|
| 平日 | 2〜3投稿 |

## 投稿バランス

投稿ジャンルの配分目安。状況に応じて変更可（災害発生時など速報優先の状況では、この配分にこだわらない）。

| ジャンル | 目安割合 |
|---|---|
| 行政・防災 | 30% |
| イベント | 25% |
| 暮らし情報 | 15% |
| 観光 | 10% |
| 市議会・財政 | 10% |
| サイト更新報告 | 5% |
| 地域交流（返信・引用など） | 5% |

## 初月運用スケジュール（4週間）

フェーズ13.2「Xアカウント初期育成」の実行計画。曜日ごとに投稿ジャンルをローテーションし、行政情報に偏らないようにする。日付は実施開始日に応じて第1週〜第4週に読み替える。実際の投稿内容（見出し・文面）はその日の状況（新着記事・イベント・天候・災害有無等）に応じて選ぶ。返信・いいね・フォローは毎日共通のルーティンとして扱う。

### 曜日別ローテーション（毎週共通・4週間繰り返す）

| 曜日 | 投稿ジャンル（1本目） | 投稿ジャンル（2本目、平日のみ） | 返信 | いいね | フォロー |
|---|---|---|---|---|---|
| 月 | 行政情報 | 新着記事（あれば） | ★★★★★アカウントの投稿に1件 | 3〜5件 | 週合計5〜10件のペースで1〜2件 |
| 火 | 防災 または ライブカメラ | 暮らし情報 | 公共施設・学校の投稿に1件 | 3〜5件 | 1〜2件 |
| 水 | 市議会（定例会期中）または財政ウォッチ | 新着記事（あれば） | 地域店舗・文化団体の投稿に1件 | 3〜5件 | 1〜2件 |
| 木 | 観光・文化 | 交通・くらしガイド系 | 観光協会・文化施設の投稿に1件 | 3〜5件 | 1〜2件 |
| 金 | イベント告知 | 新着記事（あれば） | 地域イベント主催者の投稿に1件 | 3〜5件 | 1〜2件 |
| 土 | イベント／高校受験（季節による） | － | 市民・地域アカウントの投稿に1件 | 3〜5件 | 0〜1件 |
| 日 | 週まとめ・人気記事再掲、またはお知らせ | － | 週内で交流できなかった対象に1件 | 3〜5件 | 0〜1件 |

### 週ごとの重点（4週間）

| 週 | 重点テーマ | ねらい |
|---|---|---|
| 第1週 | 自己紹介・アカウントの案内、行政情報・防災の基本投稿を中心に | フォロワーに「何のアカウントか」を認知してもらう |
| 第2週 | イベント情報・ライブカメラ・観光文化を積極的に投稿 | 行政情報だけに偏らないバランスを見せる |
| 第3週 | 財政ウォッチ・市議会ウォッチ・高校受験ガイド等、差別化コンテンツを紹介 | 他の地域アカウントにはない専門性をアピール |
| 第4週 | 月間の振り返り投稿、人気記事の再掲、来月の予告 | 月次レビュー（[7. 月次レビュー](#7-月次レビューテンプレート)）につなげる |

※ 災害・緊急情報が発生した場合は、上記のローテーションよりも速報を優先する（[3. 投稿頻度](#3-投稿頻度)の「速報は随時」ルールが優先）。

## 4. フォロー方針

- 毎週5〜10アカウントを追加
- 大量フォロー（フォロー数の急増）は禁止
- フォロー解除は基本しない
- 地域アカウント（宝塚市内・阪神間の団体・店舗・個人）を優先
- フォローしたアカウントは都度 [SNS_ACCOUNT_LIST.md](./SNS_ACCOUNT_LIST.md) に追記する

---

## 5. 返信方針

**積極的に交流する対象**
- 宝塚市公式
- 公共施設
- 地域イベント（主催者・参加者）
- 地域店舗
- 学校
- 文化団体
- 市民

**注意事項**
- 政治的議論には参加しない
- 記事生成ルール（事実のみ・誇張禁止・原文転載禁止）と同じ姿勢で、返信でも評価・意見の表明は行わない

## 投稿テンプレート

`{　}`は差し替え箇所。記事URLを含める・出典を明記する・誇張しないという[PROJECT_RULES.md](../PROJECT_RULES.md)のSNS運用方針・記事生成ルールに従う。

### 新着記事

```
【新着記事】{記事タイトル}
{記事の要点を1〜2文で}
{記事URL}
#宝塚市
```

### イベント

```
【イベント情報】{イベント名}
日時：{日時}
場所：{場所}
{イベント概要を1文で}
詳細：{記事URL または出典URL}
#宝塚市 #{イベントに関連するタグ}
```

### 行政情報

```
【行政情報】{見出し}
{制度・手続きの概要を2〜3文で}
対象：{対象者}　期限：{期限があれば}
詳細：{出典URL}
#宝塚市
```

### 防災情報

```
【防災情報】{見出し}
{内容を簡潔に。警戒レベルや対象地域があれば明記}
最新情報は宝塚市公式・気象庁等でご確認ください。
{出典URL}
#宝塚市 #防災
```

### ライブカメラ

```
【ライブカメラ】{地点名}の様子
{天候・状況の一言（分かる場合のみ）}
武庫川の水位・河川の状況はこちらから確認できます
{ライブカメラページURL}
#宝塚市 #武庫川
```

### 財政ウォッチ

```
【財政ウォッチ】宝塚市の財政状況（{公表月}）
・{ポイント1}
・{ポイント2}
くわしくはこちら
{財政ウォッチページURL}
#宝塚市 #財政ウォッチ
```

### 高校受験

```
【高校受験】{見出し}
{内容（学区・日程・確定している制度情報のみ。偏差値・倍率等の主観的情報は含めない）}
学区・高校一覧はこちら
{学区・高校受験ガイドURL}
#宝塚市 #高校受験
```

### お知らせ（サイト自体のお知らせ）

```
【お知らせ】{お知らせ内容}
{補足説明があれば1文}
{関連URL（あれば）}
#宝塚市 #TakarazukaToday
```

## 運用ルール

- フォロワー数だけを目標にしない
- 地域との信頼関係を最優先とする
- 誤情報は発信しない
- 引用元を明記する
- 政治的な議論には参加しない
- 災害時は速報性より正確性を優先する
