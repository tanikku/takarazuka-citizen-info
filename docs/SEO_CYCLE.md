## 改善優先順位の判断フロー

### ① 平均順位3〜10位 かつ CTRが期待値より低いページ → title・description改善

- 順位別のCTR期待値の目安：1位 約25〜30%／3位 約10%／5位 約6%／10位 約3%
- 期待値を大きく下回る場合、検索結果上の見え方（title・description）と検索意図のミスマッチを疑う
- 対象クエリの語順・地名がtitleに含まれているかを最初に確認する（例：「宝塚駅 喫煙所」→titleに「宝塚駅」が入っているか。2026-07-23の喫煙所ガイド改善の事例参照）

### ② 平均順位11〜20位 → 内部リンク追加、必要に応じて本文改善

- サイト内の関連ページ（資産ページ・記事）からのリンクを増やす
- `data/recommendations.json`への追加、関連ガイドのrelatedLinksへの追加を検討する
- 本文が薄い場合はセクション追加を検討する（一次情報の確認を必須とする）

### ③ 20位以下 → コンテンツ追加を検討

- 検索意図に対してページの情報量・網羅性が不足している可能性が高い
- 新セクション追加、FAQ拡充、または専用ページの新設を検討する
- 新設する場合は既存ページとのカニバリを必ず先に確認する（DECISIONS.mdの「引っ越し・転入手続きガイドは新規作成せず既存ページを拡充」の判断パターン参照）

## 運用ルール

- このフローは毎月のSearch Console確認（次回目安：2026年8月上旬）とセットで実行する
- 1回の確認で実施する改善は2〜3ページまでに絞る（効果測定を切り分けるため）
- title・descriptionを変更したら、このファイルの改善履歴と[DECISIONS.md](../DECISIONS.md)の両方に記録する
- `updatedAt`は内容の変更を伴う場合のみ更新する（title変更のみでは更新しない。既存方針どおり）
