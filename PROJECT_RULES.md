# PROJECT_RULES.md — 宝塚Today 運営ルール

このファイルは「常に従うべき判断基準・ルール」を集約したもの。
実装状況は[[PROJECT_STATUS.md]]、過去の意思決定の経緯は[[DECISIONS.md]]を参照。
簡易版は`SKILL_TAKARAZUKA_TODAY.md`にも要約あり（本ファイルが詳細版）。

## サイトの目的
1. 宝塚市民が必要な行政・防災・生活情報を分かりやすく取得できること
2. 検索流入を増やし継続的に利用されること
3. 公式情報を中心に信頼性の高い地域メディアを構築すること

## 技術方針
- 静的HTML/CSS/JavaScriptで構築（データベース・管理画面は使わない）
- HTML生成は素のNode.jsスクリプト（外部SSG・フレームワーク不使用）
- Cloudflare Pagesで無料公開、月間10万PV程度まで耐えられる構成を想定
- スマホ対応（レスポンシブ）必須、モバイル優先で設計する
- SEO対応必須（meta/構造化データ/sitemap.xml）
- CMS化・データベース化は指示があるまで実装しない

## 運用ルール（最重要方針）
- **量より質を優先する。記事数目標は設定しない**
- valueScore（S/A/B/C）による公開範囲の制御：
  - **S・A：必ず公開**（トップニュース・今日のトピック・カテゴリープレビューに表示）
  - **B：一覧表示のみ**（カテゴリーアーカイブ・新着一覧。トップの目立つ枠には出さない）
  - **C：非公開**（HTML生成・サイトマップとも対象外）
- 評価基準：
  - S：税金変更／ゴミ収集変更／学校統廃合／給食費変更／大規模工事／災害情報／防犯情報／公共交通変更／市議会重要決定 など
  - A：市主催イベント／子育て支援／教育施策／公共施設開設／健康福祉制度変更／市予算の主要項目 など
  - B：特定世代向け制度／地域限定イベント／各種募集／補助金案内 など
  - C：入札公告／審議会委員募集／業者向け案内／会議開催通知／事務手続きのみ など
- 自動生成された記事は**公開前に人間レビューを必須**とする。完全自動公開には切り替えない
- ドラフト運用フロー：`fetch:all`（候補取得）→ Claudeが手動で内容確認・要約作成 → `data/drafts/`（`approved:false`） → 人間が承認 → `promote` → `data/articles/` → `build`

## 記事生成ルール
- 事実のみ記載する。推測は禁止
- 誇張した表現は禁止
- 原文の転載は禁止（要約・言い換えで記載）
- 出典URLを必須で記載する
- 文字数は300〜500文字程度
- 市民向けにわかりやすく説明する。「何が変わるか」を最優先で説明する
- ドラフト作成時の留意点：
  - 期限・日付が「今日」より過去の候補は記事化しない（鮮度を毎回確認）
  - 部署名に「宝塚」が含まれていても内容が他市の話である場合があるため本文を必ず確認する
  - 古い候補ばかり選ばず、直近の候補を優先する

## 情報ソース採用基準

### 新規ソース調査時の必須確認項目
1. robots.txt
2. 利用規約（自動取得・引用・要約掲載の可否）
3. RSS有無
4. API有無
5. 技術難易度
6. 運用コスト
7. SEO効果
8. PV期待値

### 評価ランク
- ◎ 採用（自動取得可）
- ○ 条件付き採用（一部制限あり）
- △ 手動運用のみ（自動取得は行わない）
- × 不採用

### 採用済みソース
- 宝塚市公式サイト（RSS）、兵庫県公式サイト（cheerio）、兵庫県警察（cheerio）：◎
- 宝塚市議会（DiscussVisionSmart＋gijiroku.com）：△（手動確認運用、詳細は[[DECISIONS.md]]）

### 不採用ソース
- 競合ローカルメディア（号外NET・コミパ通信等、独自取材記事の転載が必要なサイト全般）：×
- 適法な情報源が存在しない領域（交通情報の一部、飲食店開店閉店情報）：×

## 市議会運用ルール
- 市議会情報は独自性の高い差別化コンテンツ。**PV効果は限定的なため検索流入目的ではなく差別化目的で運用する**
- 速報：DiscussVisionSmartで議員の質問テーマを確認
- 詳細要約：gijiroku.comの確定会議録公開後に作成
- **禁止事項**：議員ランキング、活動量ランキング、評価スコア、政治的な優劣表現、AIによる賛否評価
- 要約は事実ベースのみ（「○○議員が質問」「市が回答」「議案が可決」等）
- 会議要約ページには出典リンク必須、「AI要約・編集確認済」バッジで生成過程を明記（詳細説明は記事下部）
- 議員ページには本人の公開発言のみ掲載（参考人・陳述人等の私人情報は掲載しない）
- 自動スクレイピングは行わない（robots.txt許可状況にかかわらず）

## SEO方針
- 重視する。対象：title／description／OGP／JSON-LD／BreadcrumbList／FAQPage
- 新規固定ページ（ガイド等）はSEO対応を前提に実装する
- 検索流入重視テーマ（優先度高）：ゴミ／子育て／防災／学校／手続き／引っ越し／医療費助成
- 優先度低（基本的に作らない）：評論／感想／ランキング／主観的コンテンツ

## SNS運用方針
- 優先順位：X → Instagram
- 自動投稿は行わない。記事公開時に生成済みの`sns.x`等を使い人間が手動投稿
- 各記事JSONに`sns: { x, instagram, youtube_shorts }`を保持。記事生成ルール（事実のみ・誇張禁止・原文転載禁止・出典明記）に従う
- **Xの文面には記事URLを必ず含める**（リンクカード表示・サイトへの誘導のため）
- 再投稿対象（ガイド系、賞味期限が長いコンテンツ）：ゴミ出しガイド／子育て支援ガイド／防災ガイド／学校情報ガイド

## 自動化方針
- **許可**：GitHub Actionsによる`fetch:all`（候補取得のみ）の定期実行
- **禁止**：
  - 利用規約が不明なサイトの定期スクレイピング
  - robots.txtで禁止されている領域へのアクセス
  - 要約作成・承認・公開（promote）の自動化（人間レビュー必須の方針）
  - ヘッドレスブラウザ（puppeteer等）の導入

## 依存ライブラリ（許可済み）
- `rss-parser`：RSS解析
- `cheerio`：HTML解析（robots.txt・利用規約を確認済みのサイトのみ対象、最小限の利用）
- 未導入：PDFテキスト抽出ライブラリ（`pdf-parse`等）。着手時に再確認
- これ以外のライブラリ追加は都度相談する

## デザイン方針
- 「宝塚らしさ」を重視：上品／落ち着き／宝塚歌劇／花のみち／武庫川／市民向け／モバイル優先
- 派手なニュースサイト風にはしない
- アイコンは自作インラインSVG（外部アイコンフォント・ライブラリ不使用）
- 背景画像に重ねる文字は半透明パネルではなく縁取り文字＋太字で視認性確保（詳細は[[DECISIONS.md]]）

## 提案時の判断基準
機能追加を提案する際は必ず以下を提示する。
- 想定PV効果
- SEO効果
- 実装コスト
- 保守コスト
- 規約リスク

その上で推奨順位を提示する。**運営者は個人運営前提のため、保守負荷の低い案を優先する**。

## 注意事項
- データベース・管理画面など、現時点で不要な仕組みを先回りして作らない
- 記事生成・更新の自動化を行う際は、生成ロジックと出力HTMLの構成を明確に分離する
- ライブラリ追加・フレームワーク導入は指示なく行わない
