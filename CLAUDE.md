# 宝塚市民向け情報サイト (CLAUDE.md)

このファイルはこのプロジェクト専用の設定です。
（グローバル設定 `~/.claude/CLAUDE.md` の内容も併せて適用されます）

## サイトの目的

宝塚市の行政・暮らし情報をまとめた、宝塚市民向け情報サイト。

## 技術方針

- **静的HTML / CSS / JavaScript** で構築する（データベース・管理画面は使わない）。
- HTML生成は**素のNode.jsスクリプト**で実装する（外部の静的サイトジェネレーター・フレームワークは使わない）。
- **Cloudflare Pages** で無料公開できる構成にする。
- **スマホ対応**（レスポンシブデザイン）必須。
- **SEO対応**必須（meta タグ、構造化データ、sitemap.xml など）。
- 記事は自動生成してHTML化する。
- 維持費ゼロ〜低額、月間10万PV程度まで耐えられる構成を想定する。
- 将来的にCMS化・データベース化を検討する可能性はあるが、現時点では実装しない（指示があるまでは静的サイト構成のまま）。

## 運用ルール

- 自動生成された記事は、当面は**公開前に人間レビューを挟む**（例：Pull Requestでの確認後にmainへマージして公開）。完全自動公開には切り替えない（指示があるまで）。
- 各情報源（市公式サイト・議会・広報たからづか・兵庫県・警察防犯情報・イベント情報）からのデータ取得を実装する前に、**利用規約・robots.txtを全ソース調査してから着手する**。調査未了のソースの実装を先行させない。

### 兵庫県・兵庫県警ソースの運用上の注意（2026-06-22追記）
- 兵庫県のプレスリリースは**県全体**が対象。部署名に「宝塚」が含まれていても、内容は他市（三田市等）の話である場合が多いため、**部署名だけで宝塚関連と判断しない**。本文を必ず確認すること
- 兵庫県警の防犯情報・統計は**PDF形式が多く**、現状のテキスト解析（WebFetch等）では内容を読み取れないことが多い。広報たからづかと同様、PDF抽出ライブラリ導入まで保留
- 候補（`data/pending/`）からドラフト化する際は、**できるだけ新しい候補を優先する**（古い候補ばかり選ぶと「更新されていないサイト」という印象を与えるため）

## データ取得方法（確定方針）

### 宝塚市公式サイト（お知らせ）
- 取得元：`https://www.city.takarazuka.hyogo.jp/news.rss`（RSS 2.0）
- 取得頻度：1日1回
- **カテゴリで事前フィルタする**：採用・入札等の定時的・事務的コンテンツは除外し、市民の暮らしに関わる情報のみ記事化する
- 差分判定：`link`のURLで重複チェック

### 広報たからづか
- 取得元：月号一覧ページ → 各号の面別PDF
- 取得頻度：月号更新の確認は毎日、実際の更新は月1回程度
- **記事化する範囲は「特集・トピックス・イベント情報」面のみ**（委員募集・入札・税金のお知らせ等の事務的な面は記事化しない）
- PDFは二次利用不可のため、テキスト抽出後にPDF自体は保存しない（一時処理のみ）

両ソース共通で、フィルタ条件（除外カテゴリ／対象面）は実装時にキーワード・面タイトルでリスト化し、運用しながら調整する。

## 依存ライブラリ（許可済み）

- `rss-parser`：お知らせ（RSS）のXML解析に使用
- `cheerio`：HTML解析用（兵庫県・兵庫県警のスクレイピングに使用）。**HTML解析用途のみ・利用は最小限・puppeteer等の重量級ブラウザは導入しない・robots.txtと利用規約を確認済みのサイトのみ対象**という条件付きで許可
- PDFテキスト抽出ライブラリ（`pdf-parse`等）は**未導入**。広報たからづか・兵庫県警PDF資料の実装着手時に改めて確認する
- これ以外のライブラリ追加は、グローバル設定のルール通り、その都度指示を確認する

## 記事ドラフト運用フロー（2026-06-22実装）

```
npm run fetch:all          # 各ソースから候補を data/pending/ に保存
  ↓（Claudeがセッション中に手動で内容確認・要約作成）
data/drafts/<slug>.json    # ドラフト（記事生成ルールに従い作成。approved: false で開始）
  ↓
npm run review             # review/index.html を生成（非公開・内部確認専用）
npm run review:serve       # http://localhost:8788 でレビューページを確認
  ↓（人間が内容を確認し、問題なければdraftファイルの approved を true に変更）
npm run promote            # approved:true のdraftを data/articles/ に移動（非承認分はファイル削除or修正）
  ↓
npm run build              # data/articles/ から公開用HTMLを生成
```

- AI（Claude）による下書き生成は**セッション中の手動作業**で行う。Claude APIキーを使った自動生成は導入していない（運用が安定し記事数増加が必要になった段階で再検討）
- `review/`ディレクトリは`.gitignore`対象（公開サイトには含まれない、ローカル確認専用）
- 完全自動公開は行わない。`approved`フラグを人間が明示的に立てるまでは`data/articles/`に移動しない

## UI/UXデザイン方針（2026-06-21刷新）

- デザインは「Yahooニュース＋地域メディア（神戸新聞NEXT等）風」（3案中の案3）を採用
- アイコンは絵文字を使わず、`scripts/templates.js`内の自作インラインSVGアイコンセット（線画・モダン）を使用。外部アイコンフォント・アイコンライブラリは導入しない
- ダークモード対応：`<html data-theme>` 属性＋`localStorage`で永続化。初期化スクリプトを`<head>`にインラインで置き、画面のちらつき（FOUC）を防止
- カテゴリーは5分類で固定：行政・くらし／防犯・安全／イベント／子育て／文化・観光
- 「今日の宝塚トピック」は当日（Asia/Tokyo基準）公開記事を自動表示
- 「人気記事ランキング」は`data/ranking.json`の`weekly`配列が5件以上になるまで非表示。未蓄積時は「新着記事」パネルで代替（実データのない人気表示は誇張・推測にあたるため）
- ローカルでの見た目確認は`npm run preview`（Node標準機能のみの簡易HTTPサーバー、`scripts/preview-server.js`）を使う。`file://`で直接HTMLを開くと絶対パス（`/css/style.css`等）が解決できずCSSが効かないため注意
- デザイン案の比較検討時に作成したモック（`mockups/`配下）は確認用。本番のテンプレートとは別管理

### 記事価値スコア（S/A/B/C）
- 全記事のJSONに`valueScore`（"S"|"A"|"B"|"C"）を付与する
- 公開範囲：**S・A＝通常公開**（トップニュース・今日のトピック・カテゴリープレビューに表示）／**B＝一覧のみ**（カテゴリーアーカイブページ・新着記事一覧には表示するが、トップの目立つ枠には出さない）／**C＝非公開**（HTML生成・サイトマップ掲載とも対象外。収集記録として`data/articles/`に残すのみ）
- 評価基準：
  - **S（最優先・即公開）**：税金変更／ゴミ収集変更／学校統廃合／給食費変更／大規模工事／災害情報／防犯情報／公共交通変更／市議会重要決定 など、市民生活に直接影響する情報
  - **A（公開）**：市主催イベント／子育て支援／教育施策／公共施設開設／健康福祉制度変更／市予算の主要項目 など、多くの市民に関係する情報
  - **B（一覧のみ）**：特定世代向け制度／地域限定イベント／各種募集／補助金案内 など、一部市民向け情報
  - **C（非公開）**：入札公告／審議会委員募集／業者向け案内／会議開催通知／事務手続きのみの記事 など、行政事務情報
- 記事数目標は**撤廃**。「Sは必ず公開、Aは公開、量より質」を優先し、1日0〜10記事になることを許容する
- 将来、記事不足が続く場合は兵庫県・兵庫県警・市議会・防災情報のソース追加（規約確認後）や、ワールドカップ日本代表戦のような全市民共通の話題的トピックの特別枠を検討する

### 宝塚市議会会議録サイト(gijiroku.com) 規約調査結果

- 利用規約：専用ページはなく、標準的な著作権表示（Copyright(c) 宝塚市議会）のみ。「二次利用禁止」等の明記はない

- 文字コードはShift_JIS。取得時に変換処理が必要
